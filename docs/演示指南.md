# 演示指南

## 演示前准备（约 5 分钟）

1. 配置 `backend/.env`：
   - `LLM_API_KEY`（必填）
   - `GITHUB_TOKEN`（建议，用于私有仓库和 PR 回评）
   - `QINIU_*`（可选，展示云端报告存储）

2. 启动服务：
   ```bash
   # 终端 1
   cd backend
   python -m uvicorn app.main:app --reload --port 8000

   # 终端 2
   cd frontend
   npm run dev
   ```

3. 浏览器打开 http://localhost:3000

## 2 分钟演示脚本

| 时间 | 动作 | 讲解要点 |
|------|------|----------|
| 0:00 | 介绍产品 | 「输入 GitHub PR，AI 自动生成 Review 报告」 |
| 0:20 | 粘贴 PR 链接，点击分析 | 支持公开 PR，也支持 Token 访问私有仓库 |
| 0:30 | 展示分析进度条 | 解析 → 拉取 → 扫描 → AI 分析，SSE 流式推送 |
| 0:50 | PR 信息卡片出现 | 展示从 GitHub 实时拉取的变更统计 |
| 1:10 | 变更摘要流式输出 | LLM 概括本次 PR 目的与影响 |
| 1:30 | 风险识别列表 | 规则引擎 + AI 双通道，标注 severity |
| 1:45 | Review 建议 | 可维护性、测试、性能等维度建议 |
| 1:55 | 下载 Markdown 报告 | 结构化报告，可归档 |
| 2:00 | （可选）七牛云链接 / PR 评论 | 体现云存储与 GitHub 集成 |

## 推荐演示 PR

选择**公开、变更适中、有讨论价值**的 PR：

- 小型 PR（5 个文件以内）：响应快，适合现场
- 中型 PR（有安全或逻辑讨论点）：更好展示风险识别

示例格式：
```
https://github.com/{owner}/{repo}/pull/{number}
```

## 答辩 Q&A 参考

**Q：为什么选 DeepSeek？**
A：OpenAI 兼容接口、中文 Review 质量好、成本低，便于比赛演示。

**Q：如何控制误报？**
A：规则引擎预筛 + LLM Prompt 约束 + 不确定项标 low severity，双通道交叉验证。

**Q：大 PR 怎么处理？**
A：按 16K 字符分块，最多 3 路并行分析，结果去重合并。

**Q：和 CodeRabbit 等产品的区别？**
A：聚焦 GitHub PR 场景，支持中文 Review、七牛云报告存储、规则引擎可定制，架构开放易扩展。

**Q：七牛云在哪里用的？**
A：分析完成后将 Markdown 报告上传 Kodo，通过 CDN 域名访问，体现云存储能力。

## Webhook 演示（加分）

1. 部署后端到公网（或使用 ngrok 暴露本地 8000 端口）
2. 在 GitHub 仓库配置 Webhook → `https://域名/api/v1/webhooks/github`
3. 新建或更新 PR，观察首页历史记录自动出现分析任务
