# AI PR Review 项目测试清单

下面按照“从基础到完整流程”的顺序测试项目，尽量保证每一步都能快速定位问题。

## 1. 启动前准备

### 1.1 检查环境变量

确认后端 `backend/.env` 已配置好，至少包括：

```text
LLM_API_KEY=你的真实密钥
DATABASE_URL=sqlite+aiosqlite:///D:/Users/DELL/Desktop/七牛云/backend/prreview.db
```

如果需要访问私有仓库，再补充：

```text
GITHUB_TOKEN=你的 GitHub Token
```

### 1.2 检查依赖

后端：

```powershell
cd "D:\Users\DELL\Desktop\七牛云\backend"
pip install -r requirements.txt
```

前端：

```powershell
cd "D:\Users\DELL\Desktop\七牛云\frontend"
npm install
```

## 2. 启动服务

## 4. 业务流程测试

### 4.1 创建评审任务

在前端首页输入一个公开 PR 链接，例如：

```text
https://github.com/python/cpython/pull/1
```

点击“开始 AI Review”后，检查：

- 是否跳转到任务详情页
- 是否开始流式显示分析过程
- 最终状态是否变成“已完成”

### 4.2 检查分析结果

确认页面是否展示：

- PR 信息
- 变更摘要
- 风险点
- 建议
- 报告下载按钮

### 4.3 测试历史记录

回到首页，检查：

- 是否出现刚刚创建的任务
- 是否能看到状态
- 是否支持分页
- 是否支持按状态筛选

### 4.4 测试幂等复用

对同一个 PR 再提交一次，检查：

- 是否复用已有任务
- 是否出现“已复用同一 PR 的进行中或近期评审任务”提示
- 是否返回 `reused=true`

## 5. 常见问题排查

### 5.1 历史记录为空

可能原因：

- 后端连到了另一个数据库文件
- `backend/.env` 没改成固定路径
- 后端启动后没重新创建任务

建议先确认：

```text
DATABASE_URL=sqlite+aiosqlite:///D:/Users/DELL/Desktop/七牛云/backend/prreview.db
```

### 5.2 PR 分析失败

可能原因：

- PR 链接不是有效的 Pull Request 链接
- GitHub 网络不可访问
- `GITHUB_TOKEN` 权限不足
- `LLM_API_KEY` 配置错误

### 5.3 前端打不开

检查：

- 前端是否成功启动
- 端口 `3000` 是否被占用
- 控制台是否有报错
